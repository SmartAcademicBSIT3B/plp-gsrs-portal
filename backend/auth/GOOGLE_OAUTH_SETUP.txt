===============================================================
GOOGLE OAUTH 2.0 SETUP GUIDE FOR THESIS/CAPSTONE SYSTEM
===============================================================

QUICK SETUP STEPS:
==================

1. CREATE GOOGLE CLOUD PROJECT
   - Go to: https://console.cloud.google.com/
   - Click "Select a Project" → "NEW PROJECT"
   - Enter Project Name: "THESIS_CAPSTONE" (or your preference)
   - Click "CREATE"

2. ENABLE GOOGLE+ API
   - In Google Cloud Console, go to "APIs & Services"
   - Click "ENABLE APIS AND SERVICES"
   - Search for "Google+ API"
   - Click on it and select "ENABLE"

3. CREATE OAUTH 2.0 CREDENTIALS
   - Go to "APIs & Services" > "Credentials"
   - Click "CREATE CREDENTIALS" > "OAuth 2.0 Client ID"
   - If prompted, configure OAuth consent screen first:
     * Choose "External" as User Type
     * Fill in App name: "THESIS/CAPSTONE System"
     * Add your email as Support Email
     * Click "SAVE AND CONTINUE"
   - Back to Credentials, click "CREATE CREDENTIALS" > "OAuth 2.0 Client ID"
   - Select "Web application"
   - Add Authorized Redirect URIs:
     → http://localhost/THESIS_CAPSTONE/auth/google_login.php
     → http://yourdomain.com/THESIS_CAPSTONE/auth/google_login.php (for production)
   - Click "CREATE"

4. GET YOUR CLIENT ID AND SECRET
   - Copy the "Client ID" and "Client Secret"
   - Keep these safe - they're sensitive!

5. UPDATE google_login.php
   - Open: c:\xampp\htdocs\THESIS_CAPSTONE\auth\google_login.php
   - Replace these lines:
     $client_id = 'YOUR_GOOGLE_CLIENT_ID_HERE';
     $client_secret = 'YOUR_GOOGLE_CLIENT_SECRET_HERE';
   - With your actual Client ID and Secret from Step 4

6. ENSURE USERS EXIST IN DATABASE
   - The Google login checks if the user's email exists in students_user table
   - Make sure to register users in your database BEFORE they try Google login
   - User status must be 'active'

7. TEST THE LOGIN
   - Go to login page: http://localhost/THESIS_CAPSTONE/login.php
   - Click "Continue with PLP Account" button
   - You should be redirected to Google login
   - After login, you'll be redirected back to the system

TROUBLESHOOTING:
================

Error: "Failed to authenticate with Google"
→ Check that Client ID and Secret are correct
→ Make sure API is enabled in Google Cloud Console
→ Verify redirect URI matches exactly

Error: "Google account not registered"
→ The email must exist in students_user table
→ Contact admin to create account before Google login

Error: "Your account is not active"
→ User status in database is not 'active'
→ Contact admin to activate the account

SECURITY NOTES:
===============
✓ Client Secret should NEVER be shared or committed to public repos
✓ Use environment variables for production
✓ Always use HTTPS in production
✓ Validate all data from Google before trusting it
✓ Keep redirect_uri consistent between Google Cloud and code

HOW IT WORKS:
=============

1. User clicks "Continue with PLP Account"
   ↓
2. loginWithGoogle() redirects to auth/google_login.php
   ↓
3. google_login.php checks for ?code parameter:
   - NO CODE: Redirects to Google OAuth for user authorization
   - HAS CODE: Exchanges code for access token
   ↓
4. With access token, fetches user email and name from Google
   ↓
5. Checks if user exists in database with that email
   ↓
6. If found and active: Creates session and redirects to dashboard
   ↓
7. If not found: Shows error and redirects to login

SESSION DATA SET AFTER LOGIN:
=============================
$_SESSION['student_id']   - User's unique ID
$_SESSION['name']         - User's name
$_SESSION['email']        - User's email
$_SESSION['login_method'] - Set to 'google'
$_SESSION['login_time']   - When they logged in

PRODUCTION CHECKLIST:
=====================
□ Use environment variables or config file (don't hardcode secrets)
□ Change redirect_uri to your production domain
□ Use HTTPS (not HTTP)
□ Test with actual Google account
□ Implement rate limiting on login attempts
□ Log all login activities
□ Keep Client Secret secure
□ Monitor for suspicious login patterns
